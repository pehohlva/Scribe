文件菜单

新建：创建一个新的未命名多信息文本文稿。

新建特殊 > 

纯文本：创建新的空白纯文本文稿。

来自夹纸板的多信息文本：创建新的多信息文本文稿并插入夹纸板的内容 (之前拷贝的文本)。

来自夹纸板的纯文本：创建新的纯文本文稿并插入夹纸板的内容 (之前拷贝的文本)。

使用当前所选部分：插入最前端文稿所选的部分到新建多信息文本文稿。

打开…:

注意：怎样关联文件类型到 Bean 使得点按文稿图标就可用 Bean 打开：在 Finder 中，选择您感兴趣的文件类型。然后在右键菜单选择显示简介 > 打开方式：Bean。然后点按 '更改全部...' 按钮就可以了。

打开最近使用的：注意：在全屏幕模式下不能工作 (是个 bug, rdar://5748616)，所以在全屏幕下这一项禁用。

存储为…：Bean 在您将带附件 (如图像) 的多信息文稿存储为不支持附件的格式时会提醒您 (这种情况下附件会被抛弃)。

恢复到已存储文件：重新载入文稿已存储的版本并抛弃当前版本。程序会提醒您未存储的更改将会丢失。

导出 > 

为带样式的 HTML：当前文稿导出为 HTML 源代码文件 (带内联 CSS 样式)，将存储到原文件同一文件夹。包含的文件夹导出后在 Finder 中打开。您不能导出未存储的文稿。

为不带样式的 HTML：当前文稿导出为 HTML 源代码文件 (不带 CSS 样式，是纯 HTML)，将存储到原文件同一文件夹。

要使导出的 .html 文件中引用的图片可用，按下列步骤：

1. 在 Bean 中打开导出的 HTML 文件 (拖动显示的图标到 Bean 图标上)。

2. 在 HTML 模式替换图像的引用 (一般写在这里: img src=) 为 HTML 文件相同目录的图像文件名 (或像这样: "graphics/logo.jpg" 如果有一个 'graphics' 目录来放置图像文件)。

3. 存储,，然后连按 HTML 文件在 Safari 中打开检查效果。

为 PDF：存储文稿为 .pdf 文件到原文件同一文件夹。当您发送文稿给他人时 PDF 提供了对原文稿最大程度的保真，但一般不能被其他人编辑。

为带图片的 RTF：图像将插入到导出的 RTF 文件中。虽然 Windows 和 Linux 用户不能打开 .rtfd 或 .bean 格式，但可以打开 RTF 格式的文件。

为带图片的 DOC：与 ‘导出 > 为带图片的 RTF’ 相同，但扩展名变为 .doc，方便多数 windows 用户。MS Word 打开 RTF 文件时会不带提醒的重命名为 .doc 扩展名。

发送到 Mail： 在 Mail 程序中将文稿中所选部分作为附件添加；.rtfd 文件会自动压缩 (因为它实际是一个文件夹而非磁盘文件)。附件的文件名会显示为附件旁边的文本以便您确认发送了正确的邮件。注意：您不能邮件发送未存储的文稿。

备份：在原文件同一文件夹创建日期戳记和按序号排序的备份副本。用于版本控制，文档，以及归档。

制作模板: 将打开的文稿通过锁定文件制作为模板。Bean 之后可以重新打开模板为未命名的文稿。您可以在 Finder 中选择文件，取消选择 Finder > 显示简介 > 锁定。

在 Finder 中显示：打开 Finder 窗口，显示当前文稿所在的文件夹。注意 Finder 在分栏模式时会在预览分栏报告文件的创建和修改日期。注意：最好关闭文稿再在 Finder 中移动或重命名文稿。

页面设置…：提供文稿页面大小和方向的控制。

打印…：显示标准 OS X 打印面板。您可以选择打印机，调整打印输出，打印为 PDF，在预览程序中打开，等等。

打印所选部分…：仅打印所选的文本。对多个文本选择也有效，并可添加分隔符，如果激活了 'Bean > 偏好设置… > 高级 > 文稿 > 打印所选部分… 分隔多个所选部分选项。
