Menu Fichier

Nouveau: crée un nouveau document RTF sans titre.

Nouveau spécial > 

texte brut: crée un nouveau document vide pour texte brut.

RTF depuis le presse-papier: crée un nouveau document vide RTF et insère le contenu du presse-papier (le texte préalablement copié).

texte brut depuis le presse-papier: crée un nouveau document vide pour texte brut et insère le contenu du presse-papier (le texte préalablement copié).

avec la sélection actuelle: insère le texte actuellement sélectionné depuis le document Bean actuel dans un nouveau document texte.

Ouvrir…: affiche un panneau permettant de sélectionner et d'ouvrir un document enregistré.

Notez: Si vous voulez associer un type de fichier avec Bean afin qu'il s'ouvre toujours avec Bean si vous double-cliquez une icône de fichier, sélectionnez un fichier de ce type dans le Finder, puis sélectionnez Rechercher > Obtenir les information > puis Ouvrir avec: Bean. Ensuite, cliquez sur le bouton «Modifier tout», puis sur «OK».

Ouvrir l'élément récent: choisissez dans la liste pour rouvrir rapidement un document; cette commande ne fonctionne pas dans le mode plein écran (un bogue, rdar://5748616).

Enregistrez sous…: Bean vous avertira si l'enregistrement d'un document RTF contenant des éléments attachés (comme des images) dans un autre format ne peut pas enregistrer les images (qui se perdent ainsi).

Revenir à la version enregistrée: recharge la version enregistrée et supprime la version actuelle. Vous serez averti-e du fait que les modifications non enregistrées seront perdues.

Exporter > 

en format HTML avec mise en forme: le document actuel est exporté en format code source HTML contenant la mise en forme CSS, qui est enregistrée dans un dossier ayant le même nom que le document original. Le dossier s'ouvre dans le Finder après l'exportation. Vous ne pouvez pas exporter un document qui n'a pas été enregistré.

en format HTML sans mise en forme: le document actuel est exporté en format code source HTML sans mise en forme (code HTML pur), qui est enregistré dans un dossier ayant le même nom que le document original.

Afin de faire fonctionner les références aux images dans le code HTML, suivez ces étapes:

1. Ouvrez le document HTML dans Bean (glissez l'icône sur celle de Bean).

2. Replacez le code de référence pour les images dans le code HTML (précédé par img src=) par les noms des fichiers d'images que vous placerez dans le même dossier que le fichier HTML (ou ainsi: images/logo.jpg s'il y a un dossier «images» qui contient les images).

3. Enregistrez-le, puis double-cliquez le fichier HTML afin de l'ouvrir dans Safari, puis vérifiez son apparence.

en format PDF: enregistre le document au format .pdf, dans le même dossier que l'original. Le format PDF assure la meilleure fidélité par rapport au document de base, si vous voulez l'envoyer à d'autres, mais il ne pourra en général pas être édité par d'autres personnes.

en format RTF (avec images): les images dans le document sont intégrées dans le fichier RTF exporté. Si les utilisateurs de Windows et Linux ne peuvent pas ouvrir les fichiers .rtfd ou .bean, ils pourront probablement ouvrir les fichiers RTF.

en format DOC (avec images): la même chose que «Exporter > en format RTF avec images», mais le suffixe est .doc, ce qui mettra la plupart des utilisateurs Windows à l'aise. MS Word ouvre les fichiers RTF renommés en .doc sans se plaindre.

Envoyer à Mail: ouvre un nouveau message dans Mail.app contenant le document actif comme document attaché; les fichiers .rtfd sont automatiquement zippés (car ils sont plutôt des dossiers que des fichiers individuels). Le nom du fichier attaché est affiché à côté du fichier attaché; il permet de vérifier que vous êtes en train d'envoyer le bon fichier. Notez que vous ne pouvez pas envoyer par courriel des documents qui n'ont pas encore été enregistrés.

Sauvegarder: crée une copie de sauvegarde munie d'une date et portant un numéro de série du document enregistré que vous êtes en train d'éditer, dans le même fichier que l'original. Utile pour comparer différentes versions, la documentation et l'archivage.

Créer un modèle: fait un modèle à partir du document ouvert, en verrouillant le fichier du document. Par la suite, Bean permet de rouvrir le modèle en tant que document sans titre. Vous pouvez déverrouiller le fichier en le sélectionnant dans le Finder, puis en décochant Finder > Lire les informations > Verrouillé.

Révéler dans le Finder: ouvre une fenêtre du Finder qui contient le document actuel (enregistré). Notez que le Finder, lorsqu'il est en mode présentation par colonnes, affiche la date de création et de modification dans la colonne de prévisualisation. Notez: il vaut mieux fermer le document dans Bean avant d'utiliser le Finder pour déplacer ou renommer le document.

Format d'impression…: permet de contrôler la taille et l'orientation de la page.

Imprimer…: affiche le panneau d'impression standard d'OS X. Depuis là, vous pouvez sélectionner l'imprimante, affiner les données à imprimer, imprimer en tant que PDF, ouvrir dans Aperçu, etc.

Imprimer la sélection…: imprime seulement le texte sélectionné. Cela fonctionne avec des sélections multiples, en ajoutant des séparateurs si vous avez sélectionné Bean> Préférences… > Avancées > Document > Imprimer la sélection… sépare les sélections multiples.
